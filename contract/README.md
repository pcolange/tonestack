# Contract: compiler ↔ engine IR

The data contract between the offline Python compiler (`circuitc`) and the real-time C++
engine. The **pydantic models in `compiler/src/circuitc/ir.py` are the single source of
truth**; everything else (the generated C++ header, the golden JSON here) is derived.

## Model

A compiled circuit is a `CircuitIR`: a name, the supported base sample rates, and an ordered
list of **stages**:

- **`biquad_param_table`** — a discretized biquad sampled across a pot axis. Carries the
  `ParamSpec` (id / min / max / skew / skew_midpoint),
  an ascending `param_axis` of physical pot values, and one `BiquadRateTable` per sample rate.
  Each row is a `BiquadSection {b0,b1,b2,a1,a2}` with `a0` normalized to 1. The engine
  interpolates linearly between rows — it never evaluates a coefficient expression at runtime.
- **`waveshaper`** — a stateless nonlinear clipper split out of the LTI network. The Tube
  Screamer uses `asinh_diode`: `U = nvt·asinh(x / (2·Is·R2))`, `R2 = r2_const + r2_pot·<pot>`,
  clamped so `|U| ≤ |x|`, summed back onto the dry signal.
- **`oversample`** — wraps an inner stage list run at `factor`× the base rate. Inner biquad
  tables are therefore discretized at `base_rate · factor`.

`ir_version` is semver; the C++ loader rejects a major-version mismatch.

## Field ↔ POD parity

`BiquadSection`'s field order (`b0, b1, b2, a1, a2`) matches the C++
`tonestack::nodes::BiquadSection` POD and the transposed Direct Form II recurrence
`BiquadNode` runs. Changing one side requires changing the other.

## Golden files (this directory)

Regenerated by `circuitc compile`; **committed** as the regression target:

- `ts9_drive.coeffs.json`, `ts9_tone.coeffs.json` — the two biquad tables.
- `ts9.ir.json` — the full module IR.

`compiler/tests/test_ts9_golden.py` recomputes coefficients and compares to these files (and
to frozen literal reference points). Regenerate deliberately — and re-commit — only when the
discretization math intentionally changes. The future Lcapy → scipy symbolic path must match
the same golden.

## Generated C++ (not here)

`circuitc compile` also emits `nodes/generated/ts9_tables.h` — `constexpr` coefficient arrays
plus table/param accessors. That header is a **build artifact** (gitignored), produced by the
CMake codegen step; no JSON is parsed on the audio path.
